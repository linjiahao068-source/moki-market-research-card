# V0.2.3 StockSymbolBadge 说明

## 为什么新增 StockSymbolBadge

V0.2.3 引入了更宽泛的股票输入和生成预览场景，股票标识不再只包含 `ORCL`、`NVDA` 这类 4 位美股 ticker，也可能包含港股数字代码、带点号或短横线的 symbol。原先手写的固定字号股票方块容易在较小容器内溢出，因此新增 `StockSymbolBadge` 统一处理股票代号展示。

## 解决的问题

- `ORCL` 等 4 位代码在旧样例卡黄色方块中可能超出边框。
- 港股数字代码如 `00700` 必须保留前导 0，不能被转成数字。
- `BABA-W`、`MEITUAN-W`、`XIAOMI-W` 等较长代码不能撑破 UI，需要通过动态字号和 `truncate` 保持卡片布局稳定。

## 组件使用位置

当前 `StockSymbolBadge` 已用于：

- Landing Page 的 Live demos / 样例研究卡入口。
- `/research-cards` 列表页，因为该页面复用 `ResearchCardPreview`。
- `/generate` 研究卡预览页，用于展示解析后的 symbol、numericCode 或安全 fallback。

## 后续规则

- 股票代码始终按字符串处理。
- 不要把 `numericCode` 转成 `Number`。
- 港股、A 股等数字代码必须保留前导 0。
- 长 symbol 需要 `truncate` 或通过动态字号缩小，不能撑破卡片布局。
- 中文名不应直接塞进小方块；应优先显示匹配到的 symbol 或 numericCode，无法确定时使用安全 fallback。
