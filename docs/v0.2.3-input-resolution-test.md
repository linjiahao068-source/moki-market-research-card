# V0.2.3 股票输入解析手动测试清单

本清单用于验证 `/generate` 的股票查询输入能力。V0.2.3 使用 mock security master，不接真实金融 API、数据库或真实 AI Agent。

## 一、英文 ticker

在首页输入框或 `/generate` 页面输入以下内容，点击生成研究卡：

- `ORCL`
- `orcl`
- `NVDA`
- `BRK.B`

预期：

- 可以提交。
- `orcl` 应标准化为 `ORCL`。
- `BRK.B` 应保留点号语义。
- 页面生成 mock 研究卡预览。

## 二、纯数字代码

输入以下内容，点击生成研究卡：

- `00700`
- `700`
- `09988`
- `9988`
- `03690`
- `1810`

预期：

- 可以提交。
- 数字代码必须保持字符串处理，不得转成 Number。
- `00700`、`09988`、`03690` 等前导 0 不应丢失。
- `700`、`9988`、`1810` 应按 mock security master 的 `codeVariants` 解析。

## 三、中文名

输入以下内容，点击生成研究卡：

- `腾讯控股`
- `阿里巴巴-W`
- `美团-W`
- `小米集团-W`
- `甲骨文`
- `英伟达`
- `特斯拉`

预期：

- 中文输入可以提交。
- 中文名仅做精确匹配。
- 不做模糊搜索、不做拼音转换、不做 AI 猜测。
- 匹配成功后生成 mock 研究卡预览。

## 四、非法输入

输入以下内容，点击生成研究卡：

- 空字符串
- `@@@`
- `TSLA!`
- `甲骨文!!!`

预期：

- 空字符串提示用户输入股票代码、Ticker 或中文名。
- 非法格式提示暂不支持该输入格式或对应错误提示。
- 不应跳转到错误页面。
- 不应导致页面崩溃。

## 五、兼容旧链接

直接访问以下链接：

- `/generate?ticker=ORCL`
- `/generate?query=腾讯控股`
- `/generate?query=00700`

预期：

- `/generate?ticker=ORCL` 仍可用。
- `/generate?query=腾讯控股` 可自动带入输入框并生成或允许生成。
- `/generate?query=00700` 可自动带入输入框并生成或允许生成。

## 六、验收标准

- 不丢失前导 0。
- 中文输入可以提交。
- 未匹配输入不会导致页面崩溃。
- `ambiguous` 不会默认选第一个。
- `ambiguous` 应提示用户选择候选项或输入更精确内容。
- `npm run lint` 通过。
- `npm run build` 通过。
