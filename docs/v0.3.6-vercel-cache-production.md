# v0.3.6 Vercel 生产化与缓存优化

## 目标

v0.3.6 聚焦生产环境稳定性，减少 Vercel 页面刷新、重复生成和外部数据抖动带来的失败率。

## 已完成

- 新增共享缓存工具：`src/lib/api/routeCache.ts`
- 为 GET 数据接口添加 CDN 缓存头：
  - `/api/basic-data`
  - `/api/earnings-snapshot`
- 为 POST 生成接口添加服务端实例内存缓存：
  - `/api/research-brief`
  - `/api/serenity-memo`
- 为四个核心 API route 添加 Vercel/Next route 配置：
  - `runtime = 'nodejs'`
  - `maxDuration`
- API 响应增加：
  - `x-moki-cache`
  - `x-moki-cache-ttl`
  - `x-moki-cache-age`
- 前端真实数据请求超时调整：
  - basic data: 8 秒
  - earnings snapshot: 25 秒
- `.env.example` 补充生产缓存变量。

## 缓存策略

GET 数据接口可以被 Vercel CDN 缓存：

```http
Cache-Control: public, s-maxage=<API_DATA_CACHE_TTL_SECONDS>, stale-while-revalidate=<API_STALE_WHILE_REVALIDATE_SECONDS>
```

POST LLM 生成接口不交给 CDN 缓存，但会在同一服务端实例内做 TTL 缓存，避免同一 facts/evidence 输入短时间重复消耗模型调用。

LLM 缓存 key 包含：

- prompt version
- provider
- model
- JSON mode
- facts/evidence 的稳定 hash

缓存 hash 会忽略 `generatedAt`、`fetchedAt` 这类易变字段，避免相同事实因为时间戳不同而无法命中。

## Vercel 环境变量

建议 Production / Preview 都配置：

```env
SEC_USER_AGENT=MokiResearchCard contact@example.com
FMP_API_KEY=

LLM_PROVIDER=ark
RESEARCH_GENERATION_ENABLED=true
LLM_TIMEOUT_MS=120000
LLM_JSON_MODE=false

ARK_API_KEY=
ARK_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
ARK_MODEL=deepseek-v4-pro-260425

API_DATA_CACHE_TTL_SECONDS=300
API_STALE_WHILE_REVALIDATE_SECONDS=600
LLM_CACHE_TTL_SECONDS=1800
ROUTE_CACHE_MAX_ENTRIES=80
```

不要把任何密钥写成 `NEXT_PUBLIC_*`。

## 本地验证

```powershell
npm.cmd run lint
npm.cmd run build
Invoke-WebRequest -Uri 'http://localhost:3000/api/basic-data?query=NVDA' -UseBasicParsing
Invoke-WebRequest -Uri 'http://localhost:3000/api/earnings-snapshot?query=NVDA' -UseBasicParsing
```

检查响应头：

- `x-moki-cache: MISS`
- 再次请求同一接口应出现 `x-moki-cache: HIT`
- GET 接口应包含 `Cache-Control: public, s-maxage=...`
