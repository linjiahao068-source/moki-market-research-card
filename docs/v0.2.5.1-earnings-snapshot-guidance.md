# V0.2.5.1 Earnings Snapshot & Guidance Compare

中文名：单季度财报快照 + 指引对比模块

## 1. 用户需求背景

V0.2.5 已经引入真实基础数据接入 MVP，可以在 `/generate` 页面展示 provider、filing、部分财务字段、warnings 和 sourceLinks。但用户反馈当前展示更像“基础数据覆盖情况”，还没有回答财报季最常见的问题：

- 这个季度营收、净利润、EPS 公布值是多少？
- 和机构平均预期相比高了还是低了？
- 和去年同期相比增长还是下降？
- 公司有没有给下季度或下财年的营收 / EPS 指引？
- 指引和机构预期相比如何？

因此 V0.2.5.1 新增 Earnings Snapshot & Guidance Compare，用于把财报单季数据和指引对比整理成更接近研究卡的结构化模块。

## 2. 为什么当前 BasicDataPanel 不够

`BasicDataPanel` 的定位是基础数据面板，适合展示：

- 数据来源
- 获取时间
- 覆盖状态
- 公司名称
- 市场
- 最近 filing
- revenue / net income / assets / cash
- warnings
- sourceLinks

但它不足以表达：

- 单季度财报 period
- actual vs estimate
- surprise amount / surprise percent
- YoY 单季度变化
- guidance vs consensus
- 指标口径说明

因此 V0.2.5.1 不继续把 `BasicDataPanel` 扩成大而全模块，而是新增独立的 `EarningsSnapshotPanel` 和 `GuidanceComparePanel`。

## 3. EarningsSnapshotData 数据结构

`EarningsSnapshotData` 位于：

```text
src/types/earnings.ts
```

核心字段：

- `provider`：数据来源，例如 `sec-edgar`、`fmp`、`mock`、`manual`、`extracted`
- `fetchedAt`：数据获取时间
- `companyName`：公司名称
- `symbol`：股票 symbol 或展示代码
- `fiscalYear`：财年
- `fiscalQuarter`：财季
- `reportDate`：报告日期
- `earningsDate`：财报发布日期
- `metrics`：营收、净利润、EPS 的 actual / estimate / surprise / YoY
- `guidance`：下季度 / 下财年的营收和 EPS 指引
- `sourceLinks`：来源链接
- `warnings`：数据缺失、口径不确定或 fallback 提示

指标类型主要包括：

- `EarningsMetricComparison`
- `GuidanceMetricComparison`

这些类型都允许字段缺失，避免数据源不完整时导致页面崩溃。

## 4. 单季度 actual 的来源

单季度 actual 可以来自：

1. SEC EDGAR companyfacts / filing 数据
2. FMP income statement / earnings 数据
3. mock fallback

SEC 的优势是来源权威，但 companyfacts 中存在年度、季度、YTD、修订数据等混合情况，因此不能简单把 latest fact 当作单季度值。

V0.2.5.1 的原则：

- 只有能确认 period 是 duration 且匹配对应 filing period 时，才作为单季度 actual。
- 不用 YTD 当单季度。
- 不用全年数据当单季度。
- 不用上一季度当同比。
- 不确定时显示 `--` 并写入 warning。

## 5. 预测值和预期差距的来源

预测值通常来自第三方数据源，例如 FMP 的 analyst estimates / earnings endpoints。

V0.2.5.1 支持：

- revenue estimate
- EPS estimate
- actual EPS，如果数据源提供
- actual vs estimate 的 surpriseAbs 和 surprisePct

注意：

- SEC 不提供机构平均预期。
- 不同数据供应商对 consensus 的统计口径可能不同。
- 如果没有 estimate，不生成“超预期 / 低于预期”的结论。
- 缺失时显示 `--`，并写入 warning。

## 6. guidance 数据的难点

公司指引通常不是标准化财务字段，来源可能包括：

- earnings press release
- earnings call transcript
- 8-K exhibits
- investor presentation
- 第三方结构化数据供应商

SEC companyfacts 不直接提供结构化 guidance。

V0.2.5.1 的原则：

- 不通过正则随意抽取公司指引。
- 没有 evidenceText 和 sourceUrl 时，不标记为 verified。
- 如果 guidance 是 extracted，必须标注“文本抽取，待复核”。
- 如果没有结构化 guidance，显示“未提取到公司指引”。

## 7. 为什么没有数据时显示 --

财报和指引数据经常存在缺失、延迟、口径不一致或供应商权限限制。

为了避免误导用户，V0.2.5.1 采用保守展示：

- 缺失字段显示 `--`
- 不自动用其他期间的数据代替
- 不用 YTD 代替单季度
- 不用去年全年代替去年同期
- 不用 mock 数据伪装成真实数据

这样可以让用户明确知道哪些字段有数据，哪些字段需要后续核查。

## 8. 指标说明弹窗内容

指标说明弹窗由：

```text
src/components/earnings/MetricExplanationModal.tsx
```

提供，包含：

1. 预测值：来自第三方机构一致预期数据源，可能因数据供应商和统计口径不同而不同。
2. 公布值：来自公司财报、SEC filing 或第三方数据源。
3. 较预期：公布值减去预测值，并除以预测值计算百分比。
4. 同比：与去年同期单季度相比，不等同于环比，也不等同于年初至今。
5. 财年：财报显示的是公司财年，不一定等于自然年。
6. EPS：不同来源可能使用 GAAP 或 non-GAAP EPS，需结合来源复核。
7. 公司指引：来自公司披露、财报电话会或文本抽取。若标记为 extracted，表示仍需人工复核。
8. 非投资建议：本模块仅用于信息整理，不构成投资建议。

## 9. 后续 V0.2.6 如何接 transcript / Agent 抽取

V0.2.6 可以在 V0.2.5.1 基础上继续扩展 transcript 和 Agent 抽取能力。

建议方向：

1. 新增 Transcript Provider
   - 获取财报电话会 transcript
   - 记录来源、时间、公司、季度

2. 新增 Guidance Extraction Agent
   - 从 transcript、press release、8-K exhibit 中抽取 guidance
   - 输出结构化 `GuidanceMetricComparison`
   - 必须带 `evidenceText` 和 `sourceUrl`

3. 增加人工复核状态
   - extracted：机器抽取，待复核
   - verified：人工或可靠结构化源确认

4. 保留 fallback
   - transcript 或 Agent 失败时，不影响研究卡生成
   - guidance 显示 `--` 和 warning

5. 合规边界
   - 不生成投资建议
   - 不把指引解读成买卖信号
   - 不给目标价
   - 不承诺预测准确性

推荐未来链路：

```text
SecurityResolution
  -> BasicCompanyData
  -> EarningsSnapshotData
  -> TranscriptData
  -> GuidanceExtractionResult
  -> ResearchCard
```

V0.2.6 的重点不是“更像交易信号”，而是让证据链和指引口径更可追踪、可复核。
