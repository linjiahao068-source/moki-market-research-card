# v0.5.0 Buy-Side Report Product UI

## Goal

v0.5.0 turns the generated output into a buy-side report product view. Previous versions established the `ResearchReport` contract, source ingestion, evidence references, and buy-side generator. This version promotes the generated buy-side report to the primary `/generate` experience.

## Completed

- Updated `/generate` page metadata and hero copy to `Buy-Side Research Report`.
- Promoted `BuySideReportPanel` into the generated preview as the primary output.
- Added a `Primary Output` header that frames the report as investment view, key debates, business quality, scenarios, monitoring plan, and source audit.
- Moved the older card-style modules into a collapsed `Supporting Research Modules` area when a buy-side report is available.
- Kept source records, chunks, review count, and schema contract visible before the user reads the buy-side report.
- Advanced the runtime report schema marker to `v0.5.0` and route cache namespace to `v0.5.0`.

## PDF Alignment

- Matches the PDF direction that the product should feel like a structured buy-side report, not a stack of information cards.
- Makes the buy-side output the first real product surface after the contract/readiness strip.
- Keeps evidence and source-readiness visible before any report section is treated as publication-ready.
- Keeps old financial, guidance, scenario, and Serenity modules available as supporting diagnostics rather than the main product.

## Still Not Done

- No automatic source discovery or remote document fetching yet.
- No Technical Structure Dashboard product UI yet.
- No Yahoo chart, K-line adapter, technical indicators, or benchmark relative strength yet.
- Buy-side report content still depends on the current ResearchReport seed, source ingestion, and LLM generation constraints.

## Next Version

v0.5.1 should implement the Technical Structure Dashboard Mock product surface:

- Add a dedicated technical dashboard area in the report output.
- Show technical readiness, indicator placeholders, key zones, scenario read-through, and source gaps.
- Keep it clearly marked as mock/adapter-pending until real K-line data is connected.
