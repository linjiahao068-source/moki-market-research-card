# v0.4.9 Research Source Ingestion Real Layer

## Goal

v0.4.9 makes source ingestion an actual product layer instead of a label on legacy evidence. The system can now accept provided filings, transcripts, news, data-provider excerpts, or manual notes, split them into source chunks, attach those chunks to evidence references, and feed the resulting source coverage into native `ResearchReport` generation.

## Completed

- Added `ResearchSourceInput`, `ResearchSourceInputFact`, and `ResearchSourceChunk` schema types.
- Added `ResearchReport.sourceIngestionState.chunks` and `ResearchSourceIngestionRecord.chunkIds`.
- Updated `ResearchCard` so generated cards can carry `sourceInputs`.
- Reworked `src/lib/research-report/sourceIngestion.ts` to support two paths:
  - legacy evidence/facts adapter for existing cards;
  - provided source ingestion for user/API supplied source text.
- Added `/api/research-sources` for source ingestion smoke tests and external callers.
- Updated `/api/research-report` so callers can pass `sources` or `sourceInputs` alongside a card seed.
- Updated the ResearchReport prompt payload with source records and source chunks.
- Added an optional `/generate` Source Ingestion input panel for pasted source excerpts.
- Updated the generated preview contract panel to show source record count and chunk count.

## PDF Alignment

- Matches the PDF direction that ResearchReport generation should be grounded in sources, not just old mock card text.
- Creates a real ingestion boundary for filings, transcripts, news, data-provider text, and manual analyst notes.
- Moves source coverage into auditable `records`, `chunks`, `evidenceReferences`, and optional fact references.
- Surfaces missing source coverage before downstream buy-side and technical report layers rely on it.

## Still Not Done

- No automatic web/source discovery yet.
- No remote document fetching, PDF parsing, transcript search, or chunk retrieval index yet.
- No Yahoo chart, K-line adapter, technical indicators, or benchmark relative strength yet.
- Provided source text is accepted only when the user/API supplies it directly.

## Next Version

v0.5.0 should move the product UI toward a buy-side report experience:

- Render the generated buy-side report as the primary output, not just a schema contract.
- Promote investment view, key debates, business quality, scenario lanes, monitoring plan, and source attribution.
- Keep evidence/missing-source state visible before any report section is treated as publication-ready.
