# v0.3.7 Report Pivot Cleanup

## 目标

v0.3.7 将生成页从“多种信息卡类型”收敛为单一 `Executive Investment View`，清理旧 mock / 旧版本文案，并为 v0.4.0 `ResearchReport` schema 做过渡准备。

## 已完成

- `/generate` 顶部改为 Executive Investment View 占位结构。
- 移除用户可选卡片类型入口，内部默认生成 `executive-investment-view`。
- 删除旧 `CardTypeSelector` 组件。
- 生成页、样例库、详情页不再展示内部 `cardType`。
- 用户界面不再展示置信度百分比或证据评分圆点。
- 公司指引面板保留来源、证据数和诊断数。
- 旧 `V0.2.x mock`、`sample/mock`、`mock generator` 外显文案改为 fallback / 待补齐 / 待复核表述。
- `docs/product.md` 与 `docs/research-card-spec.md` 更新为美股研究报告过渡定位。

## 保留项

- `confidence` 仍保留在 facts、evidence、ResearchBrief 和 SerenityMemo 的内部数据结构中，用于排序、筛选和 LLM 输入。
- `ResearchCard` 仍是当前兼容类型，v0.4.0 再新增或迁移到 `ResearchReport` schema。
- 数据层 provider 中的 `mock` 类型仍作为 fallback 状态存在，本版本只清理用户可见产品文案。

## 后续交接给 v0.4.0

- 定义 `ResearchReport` 顶层 schema。
- 明确 `reportType`、`sections`、`evidenceReferences`、`sourceIngestionState` 等字段。
- 决定内部 `confidence` 是否改名为更适合用户理解的 `sourceQuality` / `evidenceWeight` / `extractionQuality`。
- 将 legacy `ResearchCard` 渲染层逐步迁移到 ResearchReport 渲染层。
