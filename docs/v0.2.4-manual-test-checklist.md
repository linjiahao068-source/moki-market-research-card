# V0.2.4 手动测试清单

本清单用于验证 V0.2.4 股票主数据与输入解析增强版。测试重点是 `/generate` 页面是否能正确处理股票代码 / Ticker / 中文名，并稳定生成 mock 研究卡预览。

## 一、英文 symbol

在 `/generate` 页面输入以下内容并点击生成：

- `ORCL`
- `orcl`
- `NVDA`
- `BRK.B`
- `BABA-W`
- `AMD`

预期：

- 合法 symbol 可以提交。
- `orcl` 应标准化为 `ORCL`。
- 带点号或短横线的 symbol 不应导致页面崩溃。
- 能生成 mock 研究卡预览或合理 fallback。

## 二、港股数字代码

输入以下内容并点击生成：

- `00700`
- `700`
- `09988`
- `9988`
- `03690`
- `3690`
- `01810`
- `1810`

预期：

- 数字代码按字符串处理。
- 不使用 `Number()` 或 `parseInt()`。
- `00700`、`09988`、`03690`、`01810` 不丢失前导 0。
- `700`、`9988`、`3690`、`1810` 应通过 `codeVariants` 匹配到对应港股。

## 三、中文名

输入以下内容并点击生成：

- `腾讯控股`
- `阿里巴巴-W`
- `美团-W`
- `小米集团-W`
- `甲骨文`
- `英伟达`
- `特斯拉`
- `微软`
- `亚马逊`
- `谷歌`

预期：

- 中文名可以提交。
- 中文名只做精确匹配。
- 不做 fuzzy search。
- 不做拼音转换。
- 不做 AI 猜测。
- 匹配成功后生成 mock 研究卡预览。

## 四、非法输入

输入以下内容并点击生成：

- 空字符串
- `@@@`
- `TSLA!`
- `甲骨文!!!`
- 空格

预期：

- 空字符串或空格提示用户输入股票代码 / Ticker / 中文名。
- 非法格式提示暂不支持该输入格式或其他明确错误提示。
- 不应跳转到错误页面。
- 不应导致页面崩溃。
- 不应生成误导性研究卡。

## 五、兼容旧链接

直接访问以下链接：

- `/generate?ticker=ORCL`
- `/generate?query=ORCL`
- `/generate?query=00700`
- `/generate?query=腾讯控股`

预期：

- `/generate?ticker=ORCL` 仍可用。
- `/generate?query=ORCL` 可用。
- `/generate?query=00700` 可用，且不丢失前导 0。
- `/generate?query=腾讯控股` 可用。

## 六、验收标准

- 不丢失 `00700` 前导 0。
- 中文名可以提交。
- 未匹配输入不会导致页面崩溃。
- `ambiguous` 不会默认选第一个。
- `ambiguous` 应显示候选项并等待用户选择。
- `StockSymbolBadge` 不溢出。
- `npm run lint` 通过。
- `npm run build` 通过。
