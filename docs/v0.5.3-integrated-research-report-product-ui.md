# v0.5.3 Integrated Research Report Product UI

v0.5.3 promotes Integrated Research Report from a supporting detail panel into the primary `/generate` product output. It unifies the buy-side report, technical K-line dashboard, evidence state, source ingestion, readiness, and review queue into one auditable report entry point.

## Implemented

- Advanced `ResearchReport.schemaVersion`, route cache key, and ResearchReport prompt cache key to `v0.5.3`.
- Added technical chart audit fields to `IntegratedReportSourceAudit`.
- Updated `integratedReportBuilder` so the technical pillar includes K-line chart availability, bar count, interval, and data-as-of.
- Added a review queue item when the technical K-line chart payload is missing.
- Enhanced `IntegratedResearchReportPanel` with readiness strip, K-line audit, technical data-as-of, and source summary chips.
- Added Integrated Research Report as the primary `/generate` output before Buy-Side and Technical report pillars.

## Planning Match

- Matches the planned final integration step: source ingestion, evidence layer, buy-side report, technical dashboard, and review queue now converge into one report surface.
- Keeps ResearchReport as the product contract rather than returning to legacy ResearchCard modules.
- Uses the v0.5.2-hotfix Yahoo K-line chart state inside the integrated source audit instead of treating technical data as a separate island.
- Preserves investment-product boundaries: no target price, rating, trading order, or recommendation.

## Still Not Included

- No automatic source discovery or filing/transcript crawler.
- No full report export or PDF generation.
- No multi-provider technical data failover beyond the current Yahoo chart path.
- No interactive review workflow for closing review queue items.
