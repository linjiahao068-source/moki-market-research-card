# v0.4.6 Integrated Research Report

## 目标

v0.4.6 在 Research Source Ingestion、Evidence Reference Layer、Buy-Side Report Generator 和 Technical Data Adapter 之上新增 Integrated Research Report。当前版本把已有研究层聚合为一个可审计的总览入口，不新增外部数据抓取、不调用 LLM、不生成评级、目标价或交易建议。

## 已完成

- 将 `ResearchReport.schemaVersion` 更新为 `v0.4.6`。
- 新增 `ResearchReport.integratedReport`，包含 readiness、pillar summaries、review queue、source audit 和 executive narrative。
- 新增 `src/lib/research-report/integratedReportBuilder.ts`，聚合 `sourceIngestionState`、`evidenceLayer`、`buySideReport`、`technicalDashboard` 和 follow-up research。
- `buildResearchReportFromCard` 构建顺序更新为 evidence layer -> buy-side report -> technical dashboard -> integrated report。
- 详情页新增 `IntegratedResearchReportPanel`，放在 Executive Summary 之后，作为整合研究报告入口。
- 侧边导航和章节编号更新为 Integrated Report 优先的结构。

## 整合原则

- Integrated Report 只读取既有 report 产物，不补写事实。
- readiness 由 source ingestion、evidence missing references、buy-side generation state 和 technical adapter state 共同决定。
- review queue 只列出需要补来源、复核 evidence 或检查 adapter 的任务。
- source audit 明确记录 source record、evidence/fact reference、linked target、missing reference 和 technical provider 状态。
- 免责声明继续说明它不是投资建议、评级或交易指令。

## 当前未做

- 不接入实时行情 provider。
- 不接入外部 SEC/PDF/news/social/call transcript 抓取。
- 不调用 LLM 生成长报告正文。
- 不生成 target price、buy/sell/hold rating、买卖点或仓位建议。
- 不迁移 `/generate` 动态链路。

## 后续交接

后续若继续产品化，应优先处理：

- 把 `/generate` 动态链路迁移到 `ResearchReport` 输出。
- 将真实 source ingestion provider 接入 `sourceIngestionState.records`。
- 将真实 technical provider 接入 `technicalDashboard.dataSnapshot`。
- 为 `integratedReport.reviewQueue` 增加可交互的补数/复核 workflow。
