# V0.2.5.1 手动测试清单

本清单用于验证 V0.2.5.1：Earnings Snapshot & Guidance Compare MVP（单季度财报快照 + 指引对比模块）。

## 一、单季度财报快照

访问以下页面并生成研究卡：

- `/generate?query=SNOW`
- `/generate?query=DELL`
- `/generate?query=NVDA`
- `/generate?query=ORCL`
- `/generate?query=AAPL`
- `/generate?query=00700`
- `/generate?query=09988`

检查：

- 是否显示财报快照模块。
- 是否显示营收、净利润、EPS。
- 是否显示预测值。
- 缺失预测值时是否显示 `--`。
- 是否显示较预期。
- 是否显示同比。
- 同比不能用 YTD。
- 同比不能用上一季度。
- 如果无法确认单季度口径，应显示缺失或 warning，不应强行展示错误期间数据。
- 财报快照不应全为 `--`，至少在有 actual 数据时展示实际值。
- 预测值缺失时可以显示 `--`。
- actual 有值时必须展示，不能因为 estimate 缺失而隐藏。

## 二、公司指引

检查：

- 如果有 guidance，是否显示下季度营收 / EPS。
- 如果刚好是财年结束，是否显示下财年完整指引。
- 如果没有 guidance 但有 evidence，是否显示“发现指引相关线索”。
- 如果没有 guidance 和 evidence，是否显示：

```text
当前未提取到结构化公司指引。
```

- `extracted` 指引必须显示“文本抽取，待复核”。
- 没有 `evidenceText` / `sourceUrl` 时，不应显示“已验证”。

## 三、指标说明

检查：

- 点击“指标说明”按钮能打开弹窗。
- 弹窗说明以下内容：
  - 预测值
  - 公布值
  - 较预期
  - 同比
  - 财年
  - EPS
  - 公司指引
  - 非投资建议
- 移动端可读。
- 弹窗或 bottom sheet 可以关闭。

## 四、基础数据面板

检查：

- `BasicDataPanel` 仍然保留。
- 它位于财报快照 / 指引模块下方，作为数据来源和 SEC 证据补充。
- `sourceLinks` 仍然可点击。
- 字段缺失时显示 `--`。
- warnings 使用温和提示，不使用强错误视觉。

## 五、构建

运行：

```powershell
npm.cmd run lint
npm.cmd run build
```

预期：

- `npm run lint` 通过。
- `npm run build` 通过。
- 旧页面仍可访问：
  - `/`
  - `/generate`
  - `/research-cards`
  - `/research-card/[slug]`
