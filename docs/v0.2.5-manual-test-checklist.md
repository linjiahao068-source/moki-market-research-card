# V0.2.5 手动测试清单

本清单用于验证 V0.2.5 真实基础数据接入 MVP。重点检查 `/generate` 是否能优先尝试基础数据接入，并在失败或不支持市场时稳定 fallback 到 mock。

## 一、美股真实基础数据

访问以下页面并点击生成研究卡：

- `/generate?query=ORCL`
- `/generate?query=NVDA`
- `/generate?query=MSFT`
- `/generate?query=AMD`

预期：

- 页面不崩溃。
- 能解析为美股 security。
- 优先尝试 SEC EDGAR Provider。
- 如果配置了 FMP，则 SEC 失败后可尝试 FMP。
- 如果真实基础数据不可用，应 fallback 到 mock，并显示对应 warning。
- 研究卡仍能生成。

## 二、SEC fallback

测试项：

- 输入不存在的美股 symbol，例如 `/generate?query=NOTREALXYZ`。
- 模拟或观察 SEC 请求失败场景。

预期：

- 页面不崩溃。
- 不显示错误 stack。
- 能 fallback 到 mock basic data。
- 能继续生成通用 mock 研究卡。
- warnings 应提示基础数据不可用或已使用 mock fallback。

## 三、港股 fallback

访问以下页面并点击生成研究卡：

- `/generate?query=00700`
- `/generate?query=腾讯控股`

预期：

- 可以解析到港股 security。
- V0.2.5 暂不请求港股真实基础数据。
- 使用 mock basic data fallback。
- warnings 包含：

```text
V0.2.5 暂未接入该市场真实基础数据。
```

- 研究卡仍能生成。

## 四、旧功能兼容

检查以下页面：

- `/generate?ticker=ORCL`
- `/research-cards`
- `/research-card/[slug]`
- 首页 Sample Cards

建议具体路径：

- `/generate?ticker=ORCL`
- `/research-cards`
- `/research-card/orcl-ai-cloud`
- `/research-card/nvda-sentiment-radar`
- `/research-card/tsla-news-decode`
- `/`

预期：

- 旧 `ticker` 参数仍兼容。
- `/research-cards` 正常展示样例库。
- `/research-card/[slug]` 详情页正常展示。
- 首页 Sample Cards 仍读取 `researchCards`，不受生成页影响。

## 五、数据展示

在 `/generate` 生成结果中检查 BasicDataPanel：

- 显示 `provider`
- 显示 `fetchedAt`
- 显示 `latestFiling`
- 显示 `sourceLinks`
- 字段缺失时显示 `--`
- warnings 能正常显示

预期：

- 数据面板移动端可读。
- sourceLinks 可点击。
- warnings 使用温和提示，不使用强错误视觉。
- 不夸大数据准确性。
- 页面仍保留非投资建议提示。
