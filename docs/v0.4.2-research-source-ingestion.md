# v0.4.2 Research Source Ingestion

## 目标

v0.4.2 在 Evidence Reference Layer 上新增 Research Source Ingestion 本地管线，把 legacy `ResearchCard.evidence`、统一 `researchEvidence` 和 `facts` 规范化为 `ResearchReport` 可消费的 source ingestion records。当前版本不联网抓取新来源，先建立可验证、可替换的摄取契约。

## 已完成

- 将 `ResearchReport.schemaVersion` 更新为 `v0.4.2`。
- 扩展 `ResearchReport.sourceIngestionState`，新增 `method`、`lastIngestedAt` 和 `records`。
- 新增 `ResearchSourceIngestionRecord`，记录每个来源的 source id、source type、method、status、sourceUrl、时间、snippet、evidenceIds、factIds 和 warnings。
- 新增 `src/lib/research-report/sourceIngestion.ts`，统一处理 legacy evidence、research data layer evidence 和 fact references。
- `buildResearchReportFromCard` 改为消费 source ingestion 输出，再组装 sections 和 evidence layer。
- `EvidenceReferencePanel` 顶部新增 Source Ingestion 状态摘要，可检查 ingestion status、method、records、freshness、last ingested 和 source summary。

## Ingestion 状态

- `legacy_card`：来自静态 legacy `ResearchCard.evidence`。
- `research_data_layer`：来自统一 `EvidenceRecord` / `FactRecord`。
- `fallback`：未匹配或 mock/fallback 输入。
- `ingested`：来源已规范化，具备基础 source label、type、snippet 和质量状态。
- `partial`：来源已进入管线，但缺少 snippet、质量未知或带 warnings。
- `fallback`：来源来自 fallback/mock/checklist，后续必须替换为可复核来源。

## 当前未做

- 不抓取 SEC filing、PDF、新闻、社媒或财报电话会原文。
- 不新增外部 API provider。
- 不解析文档正文。
- 不生成买方长报告。
- 不把 `/generate` 动态链路完全迁移到 `ResearchReport`。

## 后续交接

v0.4.3 Buy-Side Report Generator 可以从当前字段读取：

- `sourceIngestionState.records`
- `evidenceReferences`
- `factReferences`
- `evidenceLayer.links`
- `evidenceLayer.missingReferences`

生成器必须优先使用 `ingested` / `research_data_layer` 来源；遇到 `fallback` 或 `partial` 来源时，应写成待复核或缺失项，而不是补写事实。
