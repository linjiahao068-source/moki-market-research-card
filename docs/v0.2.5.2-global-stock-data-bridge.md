# V0.2.5.2 Global Stock Data Bridge

## 1. 为什么 V0.2.5.1 的 FMP / SEC 路线失败

V0.2.5.1 初始方案主要依赖 SEC EDGAR 和 FMP：

- SEC EDGAR 适合获取美股 filing、submissions 和 XBRL companyfacts。
- FMP 适合补充 profile、quote、income statement 和部分 analyst estimate。

但在“单季度财报快照 + 指引对比”场景中，这条路线存在明显短板：

1. SEC companyfacts 不直接提供机构平均预期。
2. SEC companyfacts 不直接提供结构化公司指引。
3. SEC facts 中混合了年度、季度、YTD、修订数据，单季度选择需要非常保守。
4. FMP 依赖 API key 和 plan，部分 estimates / guidance endpoint 可能不可用。
5. FMP 缺失时，财报快照容易只剩 mock 或大量 `--`。
6. 港股 / A 股不在 SEC 覆盖范围内。

因此，仅靠 SEC + FMP 难以稳定满足中文用户对“单季度财报快照”的需求。

## 2. 为什么引入 global-stock-data

`global-stock-data` 提供了更贴近中文投资场景的数据源参考，尤其包括：

- 东财 datacenter 财报三表
- 东财 GMAININDICATOR 关键指标
- Yahoo quoteSummary analyst estimates
- Yahoo news search
- SEC EDGAR / XBRL
- 股票搜索和 secid / SECUCODE 映射

这些能力可以补足 V0.2.5.1 的数据缺口，尤其是：

- 港股财报数据
- 中文字段名
- 单季度营收、净利润、EPS
- 营收同比、EPS 同比
- 分析师预期
- 指引相关新闻和 filing 线索

Moki Market 不直接运行 `SKILL.md`，而是把其中必要端点改写为 TypeScript server-side provider。

## 7. 为什么不直接在前端运行 SKILL.md

不能直接在前端运行 `SKILL.md`，原因：

- `SKILL.md` 是 Claude Code skill 文档，不是浏览器可运行模块。
- 其中示例代码是 Python `requests`，不适合 Next.js client runtime。
- 前端运行会暴露请求逻辑和潜在密钥。
- 大量数据源请求可能触发 CORS、风控和超时。
- Moki Market 需要可控的 TypeScript server-side provider 和 fallback 机制。

因此，正确策略是：

```text
参考 SKILL.md
  -> 提取 endpoint / 字段 / 映射规则
  -> 改写为 TypeScript server-side provider
  -> 用 warnings/sourceLinks 标注来源和缺失
```

## 9. 后续如何从 Bridge 进化到 Agent 数据层

V0.2.5.2 是 Bridge 阶段：

```text
vendor/global-stock-data/SKILL.md
  -> TypeScript server-side providers
  -> EarningsSnapshotData
```

后续 Agent 数据层可以演进为：

1. Security Resolver Agent
   - 使用东财 search / Yahoo search / SEC CIK mapping 做多源证券识别

2. Financial Data Agent
   - 调用 Eastmoney / SEC / Yahoo / FMP provider
   - 输出标准化 quarterly financials

3. Earnings Interpretation Agent
   - 根据 actual / estimate / YoY / guidance evidence 生成研究问题
   - 不生成买卖建议

4. Guidance Extraction Agent
   - 从 transcript / 8-K / press release 中抽取 guidance
   - 输出 evidenceText / sourceUrl / quality
   - 人工复核前标记 extracted

5. Evidence Chain Agent
   - 统一整理来源、时间、口径、缺失字段和 warnings

推荐未来链路：

```text
SecurityResolution
  -> Global data providers
  -> EarningsSnapshotData
  -> GuidanceEvidence
  -> ResearchCard
  -> Evidence chain
```

核心原则不变：数据可追踪、口径可解释、缺失要显示，不构成投资建议。
