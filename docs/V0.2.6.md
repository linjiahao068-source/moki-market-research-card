# Moki Market V0.2.6

## 版本目标

V0.2.6 的主题是 **Earnings Expectation & Guidance Compare Cleanup**。

本版本目标是把 `/generate` 生成页从“基础数据覆盖面板”体验，调整为更贴近财报季研究场景的结构：

1. 用户先看到股票身份和研究卡标题。
2. 重点查看本期财报预期差，即 actual vs consensus。
3. 再查看公司指引和相关 evidence。
4. 最后查看投研解读、风险提示、新闻 / 舆情 / 其他模块。

本版本仍然是研究辅助和产品验证功能，不构成投资建议。

## 页面结构

V0.2.6 `/generate` 生成页的目标结构：

1. 顶部股票标题 / 股票代码
2. 财报预期差 Earnings Expectation
3. 公司指引 Guidance Compare
4. 投研解读 / 风险提示
5. 新闻 / 舆情 / 其他模块

## BasicDataPanel 下线说明

V0.2.6 不再把 `BasicDataPanel` 作为用户主界面展示。

原因：

- 用户更关注单季度财报表现和指引变化。
- `BasicDataPanel` 更像数据覆盖情况检查，不是最终研究卡的核心模块。
- 过多基础字段和 `--` 会降低移动端阅读效率。

因此，`GeneratedCardPreview` 不再渲染：

- 基础数据面板
- 最近 filing 基础数据卡片
- Revenue / Net income / Assets / Cash / Quote 基础数据卡片
- 大面积 mock fallback 或未识别市场提示

## basicData provider 保留说明

虽然 `BasicDataPanel` 从主 UI 下线，但 basicData provider 仍然保留。

保留原因：

- SEC / FMP / mock basic data 仍可作为内部数据能力。
- earnings provider 仍可能复用基础数据、filing 和 sourceLinks。
- `/api/basic-data` 可继续作为调试或 fallback API。
- 未来如果需要查看数据覆盖情况，可以重新以辅助面板形式接入。

不应删除：

- `src/types/basic-data.ts`
- `src/app/api/basic-data/route.ts`
- `src/lib/dataProviders/getBasicCompanyData.ts`
- `src/lib/dataProviders/sec/*`
- `src/lib/dataProviders/fmp/*`
- `src/lib/dataProviders/mockBasicDataProvider.ts`

## Earnings Expectation 职责

`EarningsSnapshotPanel` 在 V0.2.6 中承担 **Earnings Expectation / 财报预期差** 的职责。

它只展示本期 actual vs consensus：

- Revenue actual vs consensus
- EPS actual vs consensus
- Net income 仅在 actual 和 consensus 同时存在时作为辅助项展示

它可以显示：

- 公布值
- Consensus
- 差额
- 较预期百分比
- signal label，例如“明显超预期 / 超预期 / 基本符合 / 低于预期 / 明显低于预期 / 数据不足”

它不应展示：

- guideLow
- guideMid
- guideHigh
- guidance consensus
- guidance evidence
- management guidance text
- next quarter guidance 明细

如果需要引用公司指引，只能用一句轻量摘要：

```text
公司指引：{overallSignal}，详见下方公司指引模块。
```

## Guidance Compare 职责

`GuidanceComparePanel` 是唯一展示公司指引明细和 evidence 的模块。

它负责展示：

- 下季度营收指引
- 下季度 EPS 指引
- 下财年营收指引
- 下财年 EPS 指引
- guidance vs consensus 差距
- 指引状态 signal
- evidence / 来源依据 / 原文摘要

如果没有结构化 guidance，但有 evidence，则显示：

```text
发现指引相关线索
```

如果没有 guidance 和 evidence，则显示：

```text
暂未获取到该公司的结构化公司指引，后续将接入财报新闻稿和电话会解析。
```

## 验收标准

1. 页面不再展示 `BasicDataPanel`。
2. Earnings Expectation 在 Guidance Compare 上方。
3. Earnings Expectation 不展示 guidance 明细。
4. Guidance Compare 展示 guidance 明细和 evidence。
5. `mockGenerateResearchCard` 不再生成基础数据快照 section。
6. `00700` 可以温和降级，不应页面崩溃。
7. `npm run lint` 通过。
8. `npm run build` 通过。

## 风险提示

- Earnings Expectation 中的 consensus / estimate 依赖第三方数据，可能缺失或口径不同。
- Guidance Compare 中的 extracted evidence 需要人工复核。
- 如果没有结构化 guidance，不应伪造。
- 不应将 actual vs consensus 解读为买卖建议。
- 不输出目标价、仓位、交易信号或收益承诺。
- 港股 / A 股真实 earnings 数据覆盖仍可能不足，需要 fallback 和 warnings。
