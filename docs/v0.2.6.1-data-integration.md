# V0.2.6.1 数据接入完整文档

## 概述

本文档详细说明 V0.2.6.1 版本中三类数据接入的实现状态、数据源、配置方式和使用方法。

## 数据分类

### 1. Company Actuals（公司实际财报数据）

#### 数据内容
- 季度营收 (`revenue`)
- 季度净利润 (`netIncome`)
- 季度稀释每股收益 (`dilutedEps`)
- 报告日期 (`reportDate`)
- 财年/财季 (`fiscalYear`/`fiscalQuarter`)

#### 当前接入的数据源

##### (a) SEC EDGAR（主要 Actuals 来源）
- **数据接口**：`secBasicDataProvider`
- **数据来源**：SEC EDGAR 10-K/10-Q  filings
- **覆盖范围**：美股
- **获取方式**：
  - `CIK` 查询
  - 解析 XBRL 事实数据
  - 提取 period-aligned 数值
- **当前限制**：
  - 仅支持单季度实际值
  - YoY 同比尚未完全实现
  - 无完整历史序列（仅最新财报）
- **数据质量**：高（官方数据）
- **更新频率**：季度更新（财报发布日）

##### (b) Eastmoney（备选 Actuals 来源）
- **数据接口**：`fetchEastmoneyQuarterFinancials`
- **数据来源**：东方财富财经数据
- **覆盖范围**：美股、港股、A股
- **当前限制**：
  - 主要作为 SEC 数据失败时的 Fallback
  - 货币单位处理需验证
- **数据质量**：中等
- **更新频率**：季度更新

##### (c) Yahoo Finance（备选 Actuals 来源）
- **数据接口**：`fetchYahooQuoteSummary`
- **数据来源**：Yahoo Finance API
- **覆盖范围**：全球股票
- **当前限制**：
  - 仅当 SEC 和 Eastmoney 都失败时使用
  - 数据质量和一致性不保证
- **数据质量**：低至中等
- **更新频率**：季度更新

#### Actuals Provider 选择优先级

```
1. SEC EDGAR
   ↓ (失败或无数据)
2. Eastmoney
   ↓ (失败或无数据)
3. Yahoo Finance
```

### 2. Sell-Side Consensus（卖方一致预期数据）

#### 数据内容
- 季度营收预期 (`revenueEstimate`)
- 季度 EPS 预期 (`epsEstimate`)
- 覆盖分析师人数 (`analystCountRevenue`/`analystCountEps`)
- 财年/财季 (`fiscalYear`/`fiscalQuarter`)

#### 当前接入的数据源

##### (a) Financial Modeling Prep (FMP)（主要 Consensus 来源）
- **数据接口**：`fetchFmpConsensusEstimates`
- **数据来源**：Financial Modeling Prep API
- **API Endpoints**：
  - `/analyst-estimates/{symbol}`
  - `/earnings-surprises/{symbol}`
- **覆盖范围**：美股主要标的
- **数据字段**：
  - `estimatedRevenueAvg`
  - `estimatedEpsAvg`
  - `estimatedRevenueHigh`/`estimatedRevenueLow`
  - `estimatedEpsHigh`/`estimatedEpsLow`
  - `numberAnalysts`
- **FMP 套餐需求**：
  - Analyst Estimates 数据需要 Starter 或更高套餐
  - Earnings Surprises 数据部分套餐可用
  - 完整点-in-time 历史需要专业套餐
- **当前限制**：
  - 仅支持最新共识预期
  - 无历史时间序列
  - 非完整季度覆盖（有时只有年度预期）
  - 不包含修正历史
- **数据质量**：高（数据来源权威）
- **更新频率**：分析师修正后更新

##### (b) Alpha Vantage（备选 Consensus 来源）
- **数据接口**：`fetchAlphaVantageConsensusEstimates`
- **数据来源**：Alpha Vantage API
- **API Endpoints**：
  - `EARNINGS_ESTIMATES`（Premium 接口）
- **覆盖范围**：美股主要标的
- **当前限制**：
  - `EARNINGS_ESTIMATES` 可能需要 Premium 订阅
  - 免费套餐 Rate Limit 严格（25次/天）
  - 仅当 FMP 无数据时作为 Fallback
- **数据质量**：中（取决于具体标的覆盖）
- **更新频率**：不明确

#### Consensus Provider 选择优先级

```
1. FMP（需配置 FMP_API_KEY）
   ↓ (失败或无数据)
2. Alpha Vantage（需配置 ALPHA_VANTAGE_API_KEY）
   ↓ (都失败)
3. 回退到原有 Yahoo/FMP 快照遗留数据（如存在）
```

#### 数据内容（V0.2.7+ 计划）

- **Provider**：占位实现，返回空
- **UI 接入**：未接入

#### V0.2.7 计划数据源

## 当前版本已完成接入

### 1. SEC Actuals（公司实际财报）

**状态**：✅ 完整实现并接入

**实现文件**：
- `src/lib/dataProviders/sec/secBasicDataProvider.ts`
- `src/lib/dataProviders/sec/secSubmissionsClient.ts`
- `src/lib/dataProviders/sec/secCompanyFactsClient.ts`
- `src/lib/earnings/secQuarterActualsProvider.ts`

**核心功能**：
- `getBasicCompanyData()`：获取 SEC 基础数据
- `buildSecQuarterActuals()`：将 SEC 数据转换为 earnings snapshot metrics
- 支持 `revenue`、`netIncome`、`dilutedEps` 提取
- 支持 `sourceLinks` 溯源

**使用条件**：
- 需要配置 `SEC_USER_AGENT`（公平访问标识）
- 仅美股支持（CIK 查询）

**配置示例**：
```env
SEC_USER_AGENT="Moki Market contact@example.com"
```

---

### 2. FMP Consensus（卖方一致预期）

**状态**：✅ 完整实现并接入

**实现文件**：
- `src/lib/expectations/providers/fmpExpectationsProvider.ts`
- `src/lib/config/earningsProviders.ts`
- `src/lib/expectations/providers/index.ts`（调度）

**核心功能**：
- `fetchFmpConsensusEstimates(ticker)`：获取 FMP 分析师预期
- 季度匹配逻辑：当前期和下一期分别选择
- 单位标准化：revenue 转换为 billion USD
- 多字段容错：处理不同套餐/API 版本的字段差异
- Source Note 标注："FMP analyst estimates"

**使用条件**：
- 需要配置 `FMP_API_KEY`
- 设置 `EXPECTATIONS_PROVIDER=fmp`（默认）
- 需要 FMP Starter 或更高套餐

**配置示例**：
```env
EXPECTATIONS_PROVIDER=fmp
FMP_API_KEY="your_fmp_api_key_here"
```

---

### 3. Alpha Vantage Fallback（备选一致预期）

**状态**：✅ 完整实现并接入

**实现文件**：
- `src/lib/expectations/providers/alphaVantageExpectationsProvider.ts`

**核心功能**：
- `fetchAlphaVantageConsensusEstimates(ticker)`：获取 Alpha Vantage 预期
- 多格式解析：兼容不同 API 响应结构
- Rate Limit 检测：检测 Note/Information 响应并静默返回空
- Source Note 标注："Alpha Vantage earnings estimates"

**使用条件**：
- 需要配置 `ALPHA_VANTAGE_API_KEY`
- 当 FMP 无数据或 API Key 未配置时，自动尝试 Alpha Vantage
- 需要 Alpha Vantage Premium 订阅（EARNINGS_ESTIMATES 接口）

**配置示例**：
```env
ALPHA_VANTAGE_API_KEY="your_alpha_vantage_api_key_here"
```

---

### 4. Manual Guidance Override（手动指引覆盖）

**状态**：✅ 完整实现并接入

**实现文件**：
- `src/lib/guidance/providers/manualGuidanceProvider.ts`
- `src/lib/earnings/guidanceProvider.ts`

**已支持 Ticker**：
- NVDA（NVIDIA）
- ORCL（Oracle）
- SNOW（Snowflake）
- DELL（Dell Technologies）

**数据字段**：
- `ticker`
- `fiscalYear` / `fiscalQuarter`
- `guidedPeriod`：`next_quarter` / `full_year`
- `metric`：`revenue` / `eps`
- `guideLow` / `guideMid` / `guideHigh`
- `unit`：`billion_usd` / `usd_share`
- `evidenceText` / `evidenceUrl`
- `provider`：`manual`
- `extractedAt`

**配置示例**：
```env
GUIDANCE_PROVIDER=manual
```

**添加新 Ticker Guidance**：
在 `manualGuidanceProvider.ts` 的 `MANUAL_GUIDANCE_DATA` 数组中添加新条目。

**格式示例**：
```typescript
{
  ticker: 'TICKER',
  fiscalYear: '2025',
  fiscalQuarter: 'Q1',
  guidedPeriod: 'next_quarter',
  metric: 'revenue',
  guideLow: 23.5,
  guideMid: 24.0,
  guideHigh: 24.5,
  unit: 'billion_usd',
  evidenceText: 'Company Q4 earnings call guidance: Revenue expected to be $24B ±2%',
  evidenceUrl: 'https://ir.company.com/earnings/call-transcripts',
  provider: 'manual',
  extractedAt: new Date().toISOString(),
}
```

## 环境变量完整说明

### 必需配置

```env
# SEC EDGAR 用户代理（必需，用于 SEC API 公平访问）
SEC_USER_AGENT="Moki Market contact@example.com"
```

### Consensus Data 配置

```env
# 预期数据主 Provider（默认：fmp）
# 可选：fmp | alpha_vantage | mock | manual | disabled
EXPECTATIONS_PROVIDER=fmp

# FMP API Key（FMP Consensus 需要）
FMP_API_KEY=""

# Alpha Vantage API Key（Alpha Vantage Fallback 需要）
ALPHA_VANTAGE_API_KEY=""
```

### Guidance Data 配置

```env
# 指引数据 Provider（默认：manual）
# 可选：manual | mock | disabled
GUIDANCE_PROVIDER=manual
```

### 调试配置

```env
# 启用 Earnings Data Debug API（生产环境默认禁用）
# DEBUG_EARNINGS_DATA=true
```

### 未来预留配置

```env
# 启用市场预期 Proxy（V0.2.7+）
ENABLE_MARKET_EXPECTATION_PROXY=false
```
