# v0.4.8 ResearchReport Generation API

## Goal

v0.4.8 moves the generation flow from a local `ResearchCard -> ResearchReport` bridge toward a native `ResearchReport` JSON contract. The legacy adapter remains available, but only as fallback when the LLM provider is disabled, unavailable, times out, or returns invalid JSON.

## Completed

- Added `/api/research-report` as the native ResearchReport generation endpoint.
- Added `src/lib/llm/researchReport.ts` to orchestrate seed creation, LLM JSON generation, validation, repair, and fallback.
- Added ResearchReport prompt and schema validator for stable section ids, evidence ids, fact ids, tone values, and no user-facing confidence/data-quality score fields.
- Updated `/generate` to call `/api/research-report` first and replace the preview report when a native result is available.
- Preserved native ResearchReport output when auxiliary `ResearchBrief` or `SerenityMemo` calls finish later, so they do not overwrite the native report with the old adapter.
- Added generation state metadata to `ResearchReport`, including method, provider, model, fallback status, and warnings.
- Updated the preview contract panel to show whether the report came from native API generation or compatibility fallback.

## PDF Alignment

- Matches the PDF direction that Moki Market should produce a structured buy-side `ResearchReport`, not just an information card.
- Converts the previous v0.4.7 bridge into a fallback path rather than the primary generation destination.
- Keeps the report section contract stable for later source ingestion, buy-side report rendering, and technical dashboard work.
- Keeps confidence/data-quality score fields out of the user-visible report UI.

## Still Not Done

- No source discovery, document fetching, chunking, or section-level retrieval yet.
- No Yahoo chart, K-line adapter, indicators, or benchmark relative strength yet.
- The Technical Structure Dashboard still uses the existing legacy technical context adapter, not live market data.
- LLM output is constrained by existing seed data and evidence references; it must not invent facts, ratings, target prices, or trading instructions.

## Next Version

v0.4.9 should build the real source ingestion layer:

- Accept source inputs or discovered source records.
- Normalize filings, transcripts, news, and data-provider records into `ResearchSourceIngestionRecord`.
- Attach chunks/snippets to evidence and fact references.
- Surface missing source coverage clearly before buy-side report generation.
