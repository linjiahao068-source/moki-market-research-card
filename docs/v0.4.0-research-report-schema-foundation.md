# v0.4.0 ResearchReport Schema Foundation

## 目标

v0.4.0 在 v0.3.7 `Executive Investment View` pivot 之后，建立新的 `ResearchReport` 领域 schema。当前版本只做 schema foundation 和 legacy 兼容映射，不替换页面渲染，不引入外部来源抓取，也不进入 Evidence Reference Layer 的 UI 展示细节。

## 已完成

- 新增 `src/types/research-report.ts`，定义 `ResearchReport` 顶层结构。
- 固定 `schemaVersion = v0.4.0`。
- 固定当前 `reportType = executive-investment-view`。
- 定义稳定 section id，覆盖执行摘要、财报与指引、情景地图、证据引用、技术背景、后续研究和免责声明。
- 定义 `sourceIngestionState`，承接 coverage、freshness、source summary 和 warnings，为后续 source ingestion 做预留。
- 定义 `evidenceReferences` 和 `factReferences`，把证据与事实从正文 claim 中拆出来，方便后续独立引用、排序和来源诊断。
- 新增 `src/lib/research-report/fromResearchCard.ts`，将现有 `ResearchCard` 映射为 `ResearchReport`。
- 新增 `src/data/researchReports.ts`，为静态报告数据提供读取入口。

## Schema 边界

`ResearchReport` 当前负责描述报告产物，不负责生成逻辑本身。它保存：

- 报告身份：`id`、`slug`、`schemaVersion`、`reportType`、`status`。
- 标的主体：ticker、公司名、市场、数字代码、中文名。
- 管理摘要：one-line、current state、key question、bull/bear case。
- 数据状态：source ingestion status、coverage、freshness、warnings。
- 结构化章节：section summary、claims、metrics、items、missingData。
- 引用层基础：evidence references、fact references。
- 后续研究：follow-up task、why it matters、关联 evidence/fact。
- 兼容链接：legacy ResearchCard slug 和 cardType。

## 当前未做

- 不改变 `/generate` 或 `/research-card/[slug]` 的渲染数据源。
- 不展示新的 evidence reference UI。
- 不接入新的 research source ingestion。
- 不生成买方长报告正文。
- 不实现 technical dashboard mock 或 technical data adapter。

## 后续交接

v0.4.1 应在当前 `ResearchReport.evidenceReferences` 和 `ResearchReport.factReferences` 基础上做 Evidence Reference Layer：

- 明确 claim 与 evidence/fact 的引用关系。
- 设计证据引用的 UI 或组件级数据契约。
- 标记 primary/supporting/context/fallback evidence。
- 暴露 missing source、fallback source、extraction warning 等诊断状态。

v0.4.2 再接入 Research Source Ingestion，把真实来源抓取、解析和质量状态写入当前 schema 预留字段。
