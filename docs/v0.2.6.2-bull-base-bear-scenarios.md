## 为什么该模块是“买方视角情景推演”，不是“真实买方预期数据”
- ✅ 本模块是框架性演示，展示“如何结构化思考不同情景”
- ✅ 目标是提供研究工具，不是提供目标价或预期
- ✅ 默认假设（EPS、multiple）是 mock/placeholder，用于演示计算逻辑
- ✅ 真实研究中，这些假设需要由研究员自行基于权威来源填充
- ❌ 本模块不构成任何投资建议
- ❌ 本模块的 target price 只用于演示公式，不是真实预期

## 数据输入

### 1. Actuals
- 已公布的财报数据
- 来源：SEC filings、公司财报
- 用于：base case 的历史参考

### 2. Consensus
- 市场普遍预期数据
- 来源：FactSet、Bloomberg、券商报告共识
- 用于：base case 的默认起点

### 3. Guidance
- 公司管理层指引
- 来源：earnings call、investor presentation、官方 guidance
- 用于：bull/base/bear 的上下限参考

### 4. Current Price
- 当前股价
- 来源：market data
- 用于：计算 implied return

### 5. Manual Assumptions
- 研究员自行调整的假设
- 来源：研究员判断
- 用于：微调情景以反映特定假设

## 计算公式

### 1. Target Price
```
targetPrice = EPSAssumption × ValuationMultiple
```
- EPS：某情景下的预期 EPS（full year）
- Valuation Multiple：该情景适用的估值倍数（如 P/E）
- 如果 EPS 或 multiple 缺失，targetPrice 返回 null

### 2. Implied Return
```
impliedReturnPercent = (targetPrice / currentPrice - 1) * 100
```
- 如果 currentPrice 缺失或为 0，返回 null
- 正值表示潜在上涨，负值表示潜在下跌

### 3. Probability-Weighted Target
```
probabilityWeightedTarget = Σ(targetPrice_i × probability_i)
```
- 只计算 targetPrice 不为 null 的情景
- 概率需要归一化，总和为 1（100%）

## Provider 优先级

### 1. Manual Scenario Provider
- 支持：NVDA、ORCL、SNOW、DELL、MU
- 数据：手动维护的 mock/placeholder 假设
- 特点：结构完整，用于演示

### 2. Rule-Based Scenario Provider
- 支持：任意有 earnings snapshot 的 ticker
- 规则：
  - Base EPS = consensus EPS → actual EPS（如无 consensus）
  - Bull EPS = base × 1.15
  - Bear EPS = base × 0.8
  - Base multiple = 25
  - Bull multiple = base × 1.15
  - Bear multiple = base × 0.75
- 概率：bull 25% / base 55% / bear 20%

### 3. Unavailable（空状态）
- 提示："暂未生成该公司的买方情景推演，后续将接入更多估值和预期数据"
- 不报错，温和降级
