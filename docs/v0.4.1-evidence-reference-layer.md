# v0.4.1 Evidence Reference Layer

## 目标

v0.4.1 在 `ResearchReport` schema foundation 上增加证据引用层，让报告中的 claim、metric、section item 和 follow-up task 都能追溯到 evidence/fact。当前版本只做引用关系、诊断状态和静态详情页展示，不接入新的外部来源抓取。

## 已完成

- 将 `ResearchReport.schemaVersion` 更新为 `v0.4.1`。
- 新增 `ResearchReport.evidenceLayer`，包含 summary、links 和 missingReferences。
- 新增 `src/lib/research-report/evidenceReferenceLayer.ts`，从 sections、evidenceReferences、factReferences、followUpResearch 自动构建引用矩阵。
- adapter `buildResearchReportFromCard` 现在会为每个静态报告生成 evidence layer。
- 新增 `src/components/research-report/EvidenceReferencePanel.tsx`，用于展示 evidence count、linked targets、fact refs、review items、每条 evidence 关联的 report target 和缺失引用。
- 静态研究卡详情页按 slug 读取 `ResearchReport`，证据区优先展示 v0.4.1 Evidence Reference Panel，缺失 report 时回退到 legacy evidence list。

## 引用层语义

- `linked`：evidence 已连接到至少一个 report target，且没有明显诊断问题。
- `needs_review`：evidence 已连接，但存在缺失 snippet、sourceQuality unknown 或其他 warning。
- `fallback_source`：evidence 来自 fallback/mock/checklist 等来源，必须复核后再用于正式研究。
- `missingReferences`：某个 claim、metric、section item 或 follow-up task 没有可解析 evidence，或引用的 evidence id 不存在。

## 当前未做

- 不抓取新的 research source。
- 不解析 PDF、SEC filing 或 earnings call 原文。
- 不生成买方长报告正文。
- 不把 `/generate` 动态生成链路整体迁移到 `ResearchReport`。
- 不展示置信度百分比。
