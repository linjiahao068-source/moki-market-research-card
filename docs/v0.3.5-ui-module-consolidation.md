# v0.3.5 信息卡 UI 精简与模块合并

## 目标

v0.3.5 将生成页信息卡从多段堆叠收敛为四个核心模块，降低首屏重复说明和旧 mock 提示的干扰。

核心模块：

- 财报快照
- 公司指引
- 买方情景推演
- Serenity Skill 框架

## 已完成

- 新增统一模块头 `ResearchModuleHeader`，集中展示来源、生成方式、更新时间、置信度和诊断提示。
- `GeneratedCardPreview` 重排为：决策摘要、LLM 研究摘要折叠、四核心模块、追踪清单与风险折叠。
- 删除顶栏重复的真实数据徽章，只保留数据模式与质量分。
- 公司指引、财报快照、情景推演和 Serenity 均有清晰空态或诊断入口。
- `Serenity Skill Memo` 不再放在额外折叠区中，而是作为第四核心模块直接展示。

## 本地验证

```powershell
npm.cmd run lint
npm.cmd run build
Invoke-WebRequest -Uri 'http://localhost:3000/generate' -UseBasicParsing
```

当前已知 lint warning 来自既有 Serenity real-data 文件的未使用变量，不是 v0.3.5 新增错误。

## Vercel 检查点

Production / Preview 环境变量至少应包含：

```env
SEC_USER_AGENT=
FMP_API_KEY=
LLM_PROVIDER=ark
RESEARCH_GENERATION_ENABLED=true
LLM_TIMEOUT_MS=120000
LLM_JSON_MODE=false
ARK_API_KEY=
ARK_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
ARK_MODEL=deepseek-v4-pro-260425
```

密钥变量不得使用 `NEXT_PUBLIC_` 前缀。
