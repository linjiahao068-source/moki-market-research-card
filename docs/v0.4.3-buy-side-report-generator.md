# v0.4.3 Buy-Side Report Generator

## 目标

v0.4.3 在 `ResearchReport` schema、Evidence Reference Layer 和 Research Source Ingestion 之上新增本地买方报告生成器。当前版本不联网抓取新数据、不调用 LLM、不输出目标价或交易建议，而是把已经进入 `ResearchReport` 的来源、事实、证据链接和缺失项整理成可审计的 buy-side research report 结构。

## 已完成

- 将 `ResearchReport.schemaVersion` 更新为 `v0.4.3`。
- 新增 `BuySideResearchReport`、`BuySideReportGenerationState`、`BuySideReportInvestmentView`、`BuySideReportScenario`、`BuySideReportMonitorItem` 和 source attribution 类型。
- 新增 `src/lib/research-report/buySideReportGenerator.ts`，从 `ResearchReport.sourceIngestionState`、`evidenceReferences`、`factReferences` 和 `evidenceLayer` 生成本地买方报告。
- `buildResearchReportFromCard` 现在先构建 evidence layer，再生成 `buySideReport`，避免绕过 v0.4.1/v0.4.2 的审计层。
- 研究卡详情页新增 `BuySideReportPanel`，展示 investment view、business quality、Bull/Base/Bear 情景、monitoring plan 和 source audit。

## 生成原则

- 优先使用 `ingested` 且来自 `research_data_layer` 或 `legacy_card` 的来源记录。
- 遇到 `fallback`、`partial`、缺失引用或 source warnings 时，将报告标记为 `partial` 或 `fallback`，并展示 review notes。
- Bull/Base/Bear 当前为 schema-level 结构化占位，不计算目标价、不生成投资评级、不声称概率模型已经接入。
- 所有输出均保留 evidence ids、fact ids、source attribution 和 missing references，方便后续人工复核。

## 当前未做

- 不接入外部 SEC/PDF/news/social/call transcript 数据源。
- 不调用 LLM 生成长文。
- 不生成 target price、buy/sell/hold rating 或交易指令。
- 不替代 v0.4.4 Technical Dashboard Mock。
- 不把 `/generate` 动态链路完全迁移到 `ResearchReport`。
