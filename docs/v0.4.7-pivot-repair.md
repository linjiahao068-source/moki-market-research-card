# v0.4.7 Pivot Repair

## Goal

v0.4.7 repairs the product direction after the v0.4.0-v0.4.6 implementation drift. The PDF plan requires Moki Market to move from a short card generator to a structured `ResearchReport` generator. This version does not add a new data provider. It makes the visible product contract match the intended direction again.

## Completed

- Updated `ResearchReport.schemaVersion` to `v0.4.7`.
- Changed `/generate` metadata and hero copy from a version-pivot placeholder to `ResearchReport` output language.
- Changed the generate form output structure from future migration wording to current `ResearchReport Schema` wording.
- Removed the visible data quality score from the generated preview.
- Added a `ResearchReport` bridge to the generated state so `/generate` can show schema version, section count, reference count, and missing reference count.
- Kept the legacy `ResearchCard -> ResearchReport` adapter as a transition input, not as the product destination.
- Updated README and the report structure spec to describe v0.4.7 as a repair step.

## PDF Alignment

- Matches the PDF principle that the user should see one main product shape: a buy-side research report, not multiple card types.
- Matches the rule that confidence or data quality scores must not be user-facing.
- Moves `/generate` toward `ResearchReport` as the output contract instead of saying ResearchReport is only a future migration.
- Makes the remaining adapter limitation explicit so the next version can replace it with native `ResearchReport` generation.

## Still Not Done

- No native `/api/research-report` yet.
- LLM still generates `ResearchBrief` and `SerenityMemo` auxiliary objects; v0.4.8 should replace this with full `ResearchReport JSON`.
- Source discovery, document fetching, chunking, and section retrieval are still not implemented.
- Technical Structure Dashboard, Yahoo chart K-line data, indicators, and benchmark relative strength are still not implemented.

## Next Version

v0.4.8 should add native `ResearchReport` generation:

- Add `/api/research-report`.
- Add a `ResearchReport` LLM prompt and validator.
- Make `/generate` call the new endpoint.
- Treat the legacy adapter as fallback only.
