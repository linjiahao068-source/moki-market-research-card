# v0.4.4 Technical Dashboard Mock

## 目标

v0.4.4 在 `ResearchReport` 和 v0.4.3 Buy-Side Report Generator 之上新增 Technical Dashboard Mock。当前版本只建立技术仪表盘的数据契约和页面展示，不接入实时价格、成交量、期权 IV、均线或外部行情 provider。

## 已完成

- 将 `ResearchReport.schemaVersion` 更新为 `v0.4.4`。
- 新增 `ResearchReport.technicalDashboard`，包含 summary、indicator matrix、key zones、scenario read-through、warnings 和 adapter readiness。
- 新增 `src/lib/research-report/technicalDashboardMock.ts`，从 `technical_context` section、`buySideReport.monitoringPlan`、`buySideReport.scenarios`、`sourceIngestionState` 和 `evidenceLayer` 生成 mock dashboard。
- `buildResearchReportFromCard` 现在按 evidence layer -> buy-side report -> technical dashboard mock 的顺序构建报告。
- 研究卡详情页的技术区新增 `TechnicalDashboardPanel`，同时保留 legacy technical context 文案作为过渡。
- `technicalContext.keyZones` 现在会映射为 `technical_context` section items，供 dashboard 展示关键区间。

## Mock 原则

- 所有 dashboard 指标都标记为 mock 或 adapter pending。
- 未绑定 evidence/fact/source 的技术指标显示为 `Review` 或 `Needs source`，不伪装成已验证数据。
- 情景联动只读取 v0.4.3 buy-side scenarios，不计算交易信号或技术概率。
- 关键区间来自 legacy `ResearchCard.technicalContext.keyZones`，后续由 v0.4.5 adapter 替换。

## 当前未做

- 不接入实时行情、历史 K 线、成交量、期权链、IV surface 或 technical indicator provider。
- 不计算均线、RSI、MACD、ATR、支撑/压力或 breakout 条件。
- 不生成交易建议、买卖点、止损位或仓位建议。
- 不迁移 `/generate` 动态链路。

## 后续交接

v0.4.5 Technical Data Adapter 应优先替换：

- `ResearchReport.technicalDashboard.summary.adapterReady`
- `ResearchReport.technicalDashboard.indicators`
- `ResearchReport.technicalDashboard.zones`
- `ResearchReport.technicalDashboard.warnings`
- `technical_context` section 的 evidence/fact/source links

adapter 接入后仍应保留 source attribution、review status 和 missing references，不应直接覆盖审计状态。
