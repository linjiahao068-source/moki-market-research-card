# V0.2.5.1 Earnings Math 测试说明

当前项目尚未配置单元测试框架。`src/lib/earnings/earningsMath.ts` 是纯函数工具，后续接入测试框架时建议覆盖以下用例。

## safePctChange

- `safePctChange(120, 100)` 应返回 `20`。
- `safePctChange(80, 100)` 应返回 `-20`。
- `safePctChange(120, -100)` 应返回 `220`，分母使用 `Math.abs(base)`。
- `safePctChange(undefined, 100)` 应返回 `undefined`。
- `safePctChange(100, undefined)` 应返回 `undefined`。
- `safePctChange(100, 0)` 应返回 `undefined`。

## calcSurprise

- `calcSurprise(120, 100)` 应返回 `surpriseAbs: 20` 和 `surprisePct: 20`。
- `calcSurprise(80, 100)` 应返回 `surpriseAbs: -20` 和 `surprisePct: -20`。
- 缺少 actual 或 estimate 时，应返回空对象。

## calcGuidanceMid

- `calcGuidanceMid(100, 120)` 应返回 `110`。
- `calcGuidanceMid(undefined, undefined, 105)` 应返回 `105`。
- 三个参数都缺失时，应返回 `undefined`。
- 如果 low/high 存在，应优先使用区间中点，而不是 singleValue。

## calcGuidanceGap

- `calcGuidanceGap(110, 100)` 应返回 `gapAbs: 10` 和 `gapPct: 10`。
- `calcGuidanceGap(90, 100)` 应返回 `gapAbs: -10` 和 `gapPct: -10`。
- 缺少 guidanceMid 或 consensus 时，应返回空对象。
- consensus 为 0 时，`gapPct` 应为 `undefined`。
